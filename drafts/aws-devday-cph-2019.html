11:10 - 12:00 | Building Infastructure as code using the AWS Cloud Development Kit (CDK)

Martin Beeby * Martin Beeby (@thebeebs) | Twitter * thebeebs - Overview

CDK docs: https://docs.aws.amazon.com/cdk/latest/guide/home.html https://github.com/aws/aws-cdk # is OSS - how about build a tool for ARM?

IaC Options: * Components # high level, AWS CDK, Pulumi * DSLs #Troposphere (py), SparkleFormation (rb), GoFormation (Go) * Declaratative # cloud formation, terraform * Scripted # low level, AWS SDK and CLI

Langs: * TS, JS * Java, C#, Py

CDK: * Constructs are modules in TS created using construct library CDK API * You can share your Construct as npm modules

npm install aws-cdk
cdk init sample-app --language typescript
npm run watch # recompile while working
cdk synth # create cloud formation template
cdk bootstrap # creates a S3 bucket for data while deploy. Creates a cloudformation changeset
cdk deploy # takes longer... 
cloudformation # check resources in portal
# Modify cdk-talk-stack.ts
cdk diff # diff of deployed and local code
cdk deploy

# Create lambda code to be deployed - e.g. some js

cdk diff # diff of deployed and local code

https://aws.amazon.com/amplify/ amplify-react-workshop https://github.com/sebsto/amplify-react-workshop

amplify push # data in cloud needed for amplify
amplify add auth # generates code for auth using AWS Cognito, IAM, API key or ?, which can either use hosted login page or in-app login.
amplify add api # graphQL API or REST incl obj mapper to DynamoDB
amplify add hosting # hosts a static web in S3 CDN 
amplify push
amplify publish (prod)

14:00 - 14:50 | 0 to 100kmh with GraphQL. Rapid API Prototyping using serverless backend in the cloud

Sebastien Stormacq * Sébastien ☁ Stormacq (@sebsto) | Twitter * sebsto - Overview

https://github.com/sebsto/reinvent2018-mob320

REST * Too much - overfetching: All objs in a graph * Too little: Chatty - fetch each entity in a graph GraphQL * Schemas * TypeSafe * Efficient - fetch from (resolve to) many DBs * A resolver can be added on each field * Easy test * Realtime - subscribe

Use AWS AppSync - creates GraphQL serverless API https://aws.amazon.com/appsync/ There are also OSS alternatives

You can put auth in serverless auth layer or you can nest queries # if owner is...
